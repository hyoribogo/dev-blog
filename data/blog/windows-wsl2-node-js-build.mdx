뜻밖의 좋은 기회로 **2024 오픈소스 컨트리뷰터 아카데미 멘티**로 선발되었다. 그것도 모르는 사람이 없을 수가 없는 오픈소스인 [Node.js](https://github.com/nodejs/node) 팀에 합격했다!

오프라인 발대식으로 모두와 인사를 나눈 뒤, 곧바로 그 주에 Node.js 빌드 환경을 구축하는 과제를 수행했다.<br />
이 글은 그 과정을 정리한 글이다.

<br />

## Node.js 빌드 환경 구축하기

깃허브에 [Node.js 빌드 문서](https://github.com/nodejs/node/blob/main/BUILDING.md#building-nodejs-on-supported-platforms)가 잘 정리되어 있다.

여기서는 OS 별로 빌드 환경을 구축하는 방법이 정리되어 있는데, 윈도우 환경에서 구축하는 방법도 따로 정리되어 있지만 멘토님께서는 WSL 환경을 권장하셔서 그 방법으로 진행해보기로 했다.

<br />

### WSL이란?

> WSL(Windows Subsystem for Linux)은 마이크로소프트가 윈도우 10과 윈도우 서버 2019에서 리눅스 이진 실행 파일을 네이티브로 실행할 수 있도록 하는 호환성 계층입니다. <br />WSL을 사용하면 리눅스용으로 작성된 명령줄 도구 및 응용 프로그램을 윈도우에서 실행할 수 있습니다.

말 그대로 WSL는 맥에서 Node.js를 빌드할 수 있는 환경을 윈도우에서도 **그대로 똑같이 구축**할 수 있도록 해주는 도구이다.

이제 WSL을 직접 설치해보자.

<br />

### WSL 설치

설치는 [마이크로소프트 WSL 공식 문서](https://learn.microsoft.com/ko-kr/windows/wsl/setup/environment)를 참고하면 된다.

1. 명령 프롬프트를 관리자 권한으로 실행한다.

2. `wsl --install` 명령어를 실행한다.

WSL를 설치하면 무조건 재부팅을 해야한다. <br />
재부팅 후에 우분투 앱이 새롭게 추가된 걸 확인할 수 있다.

<br />

### 우분투 에러

나는 재부팅을 했더니 우분투 앱에서 `0x80370102` 에러가 발생했다.<br />
역시 공식 문서에서 [에러코드들에 대한 해결법](https://learn.microsoft.com/ko-kr/windows/wsl/troubleshooting)이 잘 정리되어 있었다.

나의 경우 컴퓨터 BIOS 설정에서 가상화 기능이 꺼져있어서 발생한 문제였다.<br />
가상화 기능이 꺼져있는지 확인하는 방법은 `작업 관리자 > 성능 탭 > CPU 섹션`에서 가상화 지원을 확인하면 된다.

BIOS 환경에 들어가는 방법은 제조사마다 다 다르다.
나는 기가바이트 메인보드를 사용하고 있어서 부팅할 때 `F2` 키를 연타해 BIOS 환경에 진입할 수 있었다.

가상화 기능 활성화는 [해당 블로그 글](https://zzangprogrammer.tistory.com/248)을 보고 따라했다.

<br />

> 🤔 가상화가 뭔가요?

가상화는 하드웨어 리소스를 추상화하여 물리적인 하드웨어와 독립적으로 여러 개의 가상 시스템을 생성하고 실행할 수 있게 하는 기술이다. 이를 통해 하나의 물리적 머신에서 여러 운영 체제(OS)와 애플리케이션을 동시에 실행할 수 있다고 한다.

<br />

이제 성공적으로 우분투 앱을 실행할 수 있게 되었다.

실행하면 우분투 유저 이름과 비밀번호를 입력해야 한다.

비밀번호 입력 때 아무 것도 입력되지 않아 당황했는데, 알고보니 블라인드 처리가 되어 보이지 않는 것이었다. 입력은 잘 되고 있으니 오타가 나지 않게 주의하며 입력해야 한다.

비밀번호까지 입력하면 우분투 환경이 잘 구축된 것을 확인할 수 있다. <br />
이제 `sudo` 명령어를 사용할 수 있게 되었다.

<br />

### VSCode에서 WSL 사용하기

추가로 VSCode에서 WSL을 사용하면 더 편리하게 개발할 수 있다.
역시 공식문서에 정리된 [VSCode에서 WSL 사용하기](https://learn.microsoft.com/ko-kr/windows/wsl/tutorials/wsl-vscode)를 참고하면 된다.

그 뒤 우분투 환경에서 `code .` 명령어를 실행하면 VSCode가 실행되는 것을 확인할 수 있다.

## Node.js 빌드 및 테스트

이제 Node.js를 빌드하기 전 필요한 패키지들을 설치해보자.

```bash:Bash
sudo apt update && sudo apt upgrade
sudo apt install build-essential
sudo apt-get install python3 g++ make python3-pip
```

해당 명령어들을 하나씩 살펴보자.

- `sudo apt update && sudo apt upgrade`: 패키지 목록을 업데이트하여 최신 버전의 패키지 정보를 가져오고, 시스템에 설치된 패키지들을 최신 버전으로 업그레이드한다.
- `sudo apt install build-essential`: Node.js 소스 코드를 컴파일하는 데 필요한 필수 빌드 도구들을 설치한다. GCC 컴파일러, G++ 컴파일러, make 유틸리티 등 다양한 개발 도구가 포함되어 있다.
- `sudo apt-get install python3 g++ make python3-pip`: Node.js 빌드와 관련된 Python 3, C++ 컴파일러, make 유틸리티 및 Python 패키지 관리자(PIP)를 설치한다.

<br />
GCC, G++, make 등과 관련해서는 이후에 올릴 포스트에서 더 자세하게 기록할 예정이다.

<br />다 설치했다면 아래 명령어를 입력하면 된다.

```bash:Bash
./configure
time make -j4
```

이때 `-j` 뒤의 숫자는 CPU 코어 수에 따라 다르게 설정할 수 있다. 최대치를 확인하고 싶다면 `nproc`로 확인하면 된다. <br />
나는 12가 나와 안전하게 `make -j11`로 입력했었는데, WSL 환경이라 그런지 메모리 부족으로 빌드 도중 강제 종료되는 문제가 있었다. 그래서 8로 낮춰서 다시 빌드했다.

`time`을 입력하면 빌드 시간을 측정할 수 있다. 나는 빌드할 때는 `time`을 따로 입력하지 않아서 제대로 측정하지 못했다. 대략 20~30분 정도 소요된 것 같다.

<br />
빌드가 완료되면 테스트 코드들도 실행할 수 있다.

```bash:Bash
time make test-only
```

이것도 마찬가지로 `time`을 입력하면 테스트 시간을 측정할 수 있다.

테스트가 완료되면 다음과 같은 메시지가 출력된다.

```bash
All tests passed.
real 7m34.481s user 12m27.042s sys 2m7.491s
```

나는 모든 테스트를 통과했고 총 7분 34초가 걸렸다.

다른 멘티 몇 분에게는 빌드나 테스트에서 다양한 에러가 발생했는데, 나는 별다른 에러 없이 빌드와 테스트를 완료할 수 있었다.
